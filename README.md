# 🌐 Linux-C-Lab

## 🎯Goal
**「なぜそう動くか」を構造的に説明できる人になる。**  
**自ら課題を設定し、学び・創り・改善し続ける技術者として成長する。**

---

## 🧠 Concept

学び・探求・創造をテーマにした個人技術ラボ。
技術を“使う”だけでなく“理解し、自分で生み出す”ことを目指し、
Linux・C・ネットワーク・IoT・Web技術などを統合的に実践していく。

---

## 💡 Vision

* Linux と C 言語を軸に、基盤技術を深く理解し使いこなす。
* サーバ、ネットワーク、組込み、OS、IoTを自作で結合した総合的な開発環境を構築する。
* 単なる「資格」や「職歴」ではなく、**体系的知識 × 実践的スキル × 自主研究力**を育てる。
* 将来的には、低レイヤ技術企業で活躍できるエンジニアを目指す。

---

## 🧩 Philosophy

* **構造的理解**：「なぜそう動くのか」を説明できることを重視
* **自立的成長**：自ら課題を設定し、試行錯誤を通じて改善を続ける
* **創造的探究**：技術と好奇心を融合し、自分の手で“システムを生み出す”

ゴールは、**「10年キャリア × CS学士」同等以上の技術的議論ができるエンジニア**になること。

## 🗓️ フェーズ計画と進め方

各フェーズは以下の構成で進行します：

1. **Design Week（最初の週）**：設計書作成、目的・構成図・実装方針の整理
2. **実作業週（中間週）**：コード実装、検証、問題解決
3. **Blog Week（最終週）**：学習内容・工夫・結果をまとめ、技術ブログとして記録

### フェーズ一覧（36週想定）

* **Phase 1：Linuxサーバ構築（5〜6週）**
* **Phase 2：C言語・スネークゲーム（6〜7週）**
* **Phase 3：TCP通信・パケット解析（6〜7週）**
* **Phase 4：電子工作（8〜10週）**
* **Phase 5：自作ルータ（8〜10週）**
* **Phase 6（将来的挑戦）：自作OS / デバイスドライバ / カーネル拡張**

> ※ Phase6は生涯を通して挑戦する長期計画。現段階では進行フェーズの完了後に取り組む予定。

* **ロードマップ参照**：[docs/roadmap.md](docs/roadmap.md)

## 📝 記録ルール

1. **日々の実験ログを `logs/` に記録**

3. **フェーズ終了時に技術ブログ作成**

   * 学習内容・図・スクショ・コード例を整理
