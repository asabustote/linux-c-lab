DHCP

とは
Dynamic Host Configration Protocol
IPアドレスを端末へ自動的に割り当てる仕組みのこと。

存在意義
ネットワーク接続時の端末のIPアドレスを手動で設定する手間をなくすこと。

何をするのか
DHCPサーバにプールしているIPアドレスをDHCPクライアント（ネットワーク接続したい端末）に貸し出す。
IPアドレスの貸し出しと同時にネットワークの設定情報（DNSサーバ、デフォルトゲートウェイなどのIPアドレスなど）も配信できる。

DHCPクライアントがIPアドレスを取得するまでの流れ
1.DHCP Discover
DHCP クライアントがネットワーク接続時に同一のネットワーク内の端末ににパケットを送信することで
DHCP サーバへネットワーク情報を要求する。
(255.255.255.255宛のブロードキャストUDPを送信)

2.DHCP Offer
DHCP Discoverに対し、DHCP サーバがプールしているIPアドレスの1つを提案する。

3.DHCP Request
DHCP Offerに対し、DHCP クライアントが受け取ったIPアドレスに対し、正式に利用する旨申請する。

4.DHCP Ack
DHCP サーバがDHCP Requesetに対し、IPアドレス貸し出しの承諾メッセージを送る。
