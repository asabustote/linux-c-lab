#/var/www/html
Webページ公開に関するファイル置き場。

##var
効果にに関するファイル全体のルート的ディレクトリ
ログ、キャッシュ、メールキュー、アプリのデータなど

##/var/www
webサーバ用のデータが入っている
公開用のファイルがあるディレクトリ

##/var/www/html
公開用静的ドキュメントルートディレクトリ

Webサーバ（特に Apache や Nginx）の説明でよく出てくる
/var/www/html は、「公開するWebページの置き場所」 を指すディレクトリです。

そして各階層にはちゃんと意味があります👇

⸻

✅ /var/www/html の階層が意味するもの

1. /var
	•	「variable」＝変更されるデータ を置くためのディレクトリ
	•	ログ、キャッシュ、メールキュー、アプリのデータなど
	•	OSの標準的なディレクトリ構造の一部（FHS: Filesystem Hierarchy Standard）

→ Webサイトのデータも「更新される可能性が高いデータ」なので /var の配下に置かれる。

⸻

2. /var/www
	•	「Web サーバ用のデータ」 をまとめる場所
	•	Apache でも Nginx でも、Webコンテンツのデフォルト置き場として使われることが多い

→ 「Webに公開するファイルはこの下に入れる」という意味のディレクトリ。

⸻

3. /var/www/html
	•	公開する静的コンテンツ（HTML/CSS/JS/画像）」を置く “デフォルトのドキュメントルート”

例：
/var/www/html/index.html → http://server/ で表示される
	•	Apache のデフォルト DocumentRoot
	•	Nginx の標準設定例でもよく使われる

→ 要するに公開サイトのトップページがここ。

⸻

📌 なぜこの階層なの？

公式の理由（FHS）

FHS のルールでは：
	•	/var → 変更されるデータ
	•	/var/www → Webコンテンツの変更は頻繁なのでここ
	•	/var/www/html → Webの公開用ディレクトリをさらに明確にするため

こんな運用を想定しているため。

⸻

🔍 つまりまとめると…

階層	意味
/var	変更されるデータ全般
/var/www	Webサーバ用のデータ
/var/www/html	公開されるWebページの置き場所（DocumentRoot）


⸻

💡 補足：必ずこのパスである必要はない

Nginx や Apache の設定で自由に変えられるので、

root /home/ubuntu/myapp/public;

のようにして、独自のディレクトリを使ってもOKです。
